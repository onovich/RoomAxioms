# 《未登记现场》

**单元剧式异常现场保险调查推理游戏｜项目设定与玩法对接文档**  
版本：v1.0｜设定基线｜2026-06-24

> **不要调查异常。把除此之外的一切登记清楚。**

## 0. 文档定位与当前结论

本文档是当前项目的设定与玩法基线。后续实现应优先遵守其中的“核心公理”“完成条件”和“非目标”。氛围文案可以扩展，但不得引入隐藏解题规则。

- 项目名：暂定《未登记现场》。
- 结构：单元剧式异常现场调查，每关是一宗独立保险赔案。
- 核心玩法：固定信息、无猜测的格子推理；逐格调查所有安全区，并标记所有异常区。
- 首案：《别按那个键》，用于对接 Game Jam 主题；该案件可在正式版本中单独更名、替换或移除。
- 理赔结论：当前由程序根据已登记内容自动生成，不作为第二套玩家推理玩法。
- 异常：每案不同，不采用贯穿全作的统一黑盒设定。

## 1. 项目概述

| 项目项 | 当前定义 |
|---|---|
| 类型 | 异常怪谈／保险调查／固定信息格子推理／轻量单元剧叙事 |
| 玩家身份 | 保险公司“非常规赔案部门”的现场公估师 |
| 搭档 | 拥有“定则感知”能力的现场搭档 |
| 单局目标 | 调查全部安全区域，标记全部异常区域，提交完整现场登记图 |
| 失败条件 | 主动调查到异常区域，引发现地灾害，任务立即失败 |
| 关卡规模 | 通常为 3×3、4×4 或 5×5 |
| 美术载体 | 拟真的现场平面图、消防图、CAD 图和法证记录界面 |
| 叙事结构 | 独立案件为主，搭档关系、公司日常和关联档案构成弱连续线 |

### 1.1 一句话概念

保险调查员与能够感知现场规则的搭档，进入被异常遮蔽的事故现场；在绝不触碰异常的前提下，逐格勘察所有安全区域，登记全部人员、财产与资料。

### 1.2 核心体验

- 面对一张内容被遮蔽、但空间边界清晰的事故现场图。
- 依据搭档提供的可靠定则，判断下一处必然安全或必然异常的区域。
- 每次安全调查都会补全平面图，并可能发现尸体、财产、日志或关联资料。
- 玩家不是来战斗或收容异常，而是要将异常之外的一切查清楚。
- 完成全场勘察后，专业处置单位接管；系统生成冷静、程序化的保险结案摘要。

### 1.3 设计支柱

1. 公平推理：所有解题规则开局前公开，关卡必须能够在不猜测、不试错的情况下完成。
2. 职业驱动：调查全部安全格来自保险勘察职责，而不是额外的异常仪式。
3. 异常多样：每案更换异常概念、现场和小故事，底层交互保持稳定。
4. 克制叙事：现场文本提供氛围与小叙事，不承担隐藏格子线索。
5. 低主线压力：不要求所有案件汇聚为一个宏大阴谋，优先保证关卡迭代能力。

## 2. 世界背景：非常规赔案

异常现象并非公开常识，但政府、大型企业、专业处置单位与保险行业已经形成一套秘密协作流程。部分高风险客户会购买“非常规事故附加条款”。当事故现场出现异常实体、无法解释的人员损失及现场信息遮蔽时，案件会转交给保险公司的非常规赔案部门。

保险公司不会因为客户声称“发生了异常”便直接赔付。它仍需确认人员伤亡、财产损失、可回收标的和现场资料，并向后续处置单位提交异常坐标。

> **职业原则：不要调查异常。把除此之外的一切登记清楚。**

### 2.1 玩家部门的职责边界

| 负责 | 不负责 |
|---|---|
| 确认人员伤亡和财产损失 | 研究异常原理 |
| 勘察全部可安全进入的区域 | 测试异常反应 |
| 回收事故记录与重要资料 | 运输、收容或摧毁异常 |
| 通过推理标记异常坐标 | 与异常交战 |
| 提交完整现场登记图 | 决定专业处置单位的行动方案 |

## 3. “未登记现场”与现场信息遮蔽

部分异常事故会使现场各区域的具体内容受到信息遮蔽。遮蔽不会改变建筑结构，也不会让物品或异常在调查中移动；它只使“某个区域里究竟是什么”无法被普通手段可靠确认。

### 3.1 遮蔽的可观察表现

- 目击者记得某个房间或区域存在，却无法描述里面有什么。
- 常规扫描能够确认该区域存在物体、人员或活动迹象，却无法判断其具体内容或身份。
- 平面结构、墙体、房门和区域边界仍然可见，因此保险调查员可以划分标准勘察区。
- 对单一区域建立受控勘察后，若该区域安全，其原本固定的内容才会被确认并写入现场登记。

### 3.2 必须明确的边界

> **规则来源不是信息遮蔽。**现场定则之所以可知，是因为搭档拥有特殊能力；不是因为遮蔽“遮不住规则”，也不是异常主动泄露了规则。

- 开局前，每个格子的真实内容已经固定。
- 调查只揭示既有事实，不生成内容，也不改变现场。
- 异常位置在一次关卡尝试中固定，不会因玩家行为移动。
- 玩家标记只是工作笔记，不会影响真实内容或异常规律。

## 4. 主角与搭档

### 4.1 男主：非常规赔案现场公估师

男主没有战斗能力和超自然感知。他负责操作勘察设备、选择调查区域、登记人员与财产、回收资料、标注异常坐标，并提交现场报告。

- 专业特质：谨慎、程序化、善于从有限事实中作判断。
- 职业视角：面对怪异现场时，首先想到的是“还缺哪些记录”，而不是“如何消灭它”。
- 角色功能：承接玩家操作与保险职业视角。

### 4.2 搭档：定则员／定则感知者

搭档能够感知当前封闭现场中必然成立的“物与物之间的关系”，但无法直接看见未知区域的具体内容，也不能指出异常的绝对坐标。

> “我不知道灯在哪，也不知道那东西在哪。但我能确定，它不会出现在灯的附近。”

- 能够感知：局部数量、局部排除、邻域关系、异常总数等必然定则。
- 不能感知：某格的直接身份、异常的具体坐标、文档内容和案件责任。
- 公司设备只负责记录、校验和标准化表达，不负责产生规则。
- 所有正式显示的现场定则始终为真、精确、无隐藏例外。
- 搭档同时承担提示系统与轻量角色关系，不承担“直接报答案”的功能。

## 5. 异常的统一玩法属性

每个案件可以拥有不同的异常：物体、人物、房间、声音、植物、空间入口、重复尸体、无主座位或设备内的未知组织。它们不必共享来源、外观或行为。

### 5.1 所有可用异常必须满足

1. 位置在关卡开始前固定。
2. 常规手段无法直接识别其所在区域。
3. 主动调查异常会立即触发不可逆的现场级灾害。
4. 灾害足以终止本次任务，例如在场生命同步湮灭、现场从现实中消失、调查者存在被抹除，或异常沿勘察通道扩散到指挥区。
5. 异常位置能够通过公开定则和已揭示事实被唯一、无猜测地推断。
6. 玩家只负责定位，不负责触碰、测试或收容。

### 5.2 玩法上的明确结果

> **调查异常格 = 立即失败。**不设计污染值、精神值、可承受伤害或随机减益。玩家的唯一安全方式是先证明一个格子安全，再进行调查。

| 抽象机制 | 《未登记现场》中的表现 |

| 未知格 | 内容受到信息遮蔽、尚未定点勘察的区域 |
| 安全内容类型 | 应急灯、终端、尸体、档案柜、病床、设备、空区等 |
| 危险目标 | 本案异常所在区域 |
| 调查未知格 | 对单一区域建立受控勘察并登记内容 |
| 标记异常 | 在现场图上标注“异常坐标／禁止勘察” |
| 标记安全 | 标注“可勘察”，作为工作笔记 |
| 查看规则范围 | 高亮该定则涉及的正交域或邻接域 |
| 请求提示 | 请搭档复核当前事实下的一个必然结论 |
| 提交结论 | 提交完整现场登记图 |
| 调查到异常 | 现场级灾害发生，任务立即失败 |

## 7. 核心玩法公理

1. 所有参与解题的规则在开局前公开。
2. 每个格子的真实内容在开局前固定。
3. 调查安全格只会揭示既有事实；调查异常格立即失败。
4. 玩家标记只用于记录，不改变规则或世界状态。
5. 游戏不能要求玩家通过失败试错获取必要信息。
6. 文案双关、尸体朝向、环境音、动画细节等不得承担隐藏解题信息。
7. 不能把“作者保证唯一解”作为玩家需要使用的线索。
8. 合法关卡应保证玩家持续拥有至少一个可证明的安全格或异常格，直到完成。

## 8. 棋盘、内容与邻域

### 8.1 棋盘

- 常用规模：3×3、4×4、5×5。
- 格子在美术上可以是房间、病床区、仓位、隔间、车厢区或舱室模块。
- 格子必须具有清晰、无歧义的点击边界。
- 棋盘外的空间永远不计入任何邻域。

### 8.2 邻域定义

| 名称 | 定义 | 界面表现 |
|---|---|---|
| 正交邻域 | 与目标格共享边的上、下、左、右格子 | 十字形高亮 |
| 邻接域 | 与目标格共享边或角的八方向格子 | 3×3 周边框高亮，不含目标格 |

角落、边缘和内部格子的有效邻域数量不同，这是公开规则的一部分。

### 8.3 常用规则模板

| 模板 | 示例 |
|---|---|
| 全局数量 | 本现场恰好存在 3 个异常区域。 |
| 局部计数 | 每具尸体的正交邻域中，恰好存在 1 个损坏终端。 |
| 局部排除 | 每个应急灯的邻接域中，异常数量为 0。 |
| 物件计数 | 每个档案柜的正交邻域中，恰好存在 2 个空区域。 |
| 总量闭合 | 剩余异常数等于剩余候选格数时，候选格全部为异常。 |
| 交集推理 | 多个规则范围相交后，推出某格必然安全或异常。 |

## 9. 系统推理模型

系统内部维护“候选现场集合”。每个候选现场都是一种满足全部公开定则与已揭示事实的完整布局。随着玩家调查安全格，候选集合只能缩小，不能增加。

| 判定 | 系统定义 |
|---|---|
| 必然安全 | 该格在所有剩余候选现场中都不是异常。 |
| 必然异常 | 该格在所有剩余候选现场中都是异常。 |
| 尚不可判定 | 该格在不同候选现场中可能安全，也可能异常。 |

- 提示系统只能给出必然安全或必然异常的结论，并说明使用了哪些定则与已知事实。
- 关卡验收时，应模拟玩家可用信息，验证整个流程无须猜测。
- 若失败后允许重试，应生成新的有效布局或采用其他防试错策略，避免玩家靠记住死亡格枚举答案。

## 10. 单局完整流程

1. **接收赔案**：展示投保人、事故地点、申报损失、已知伤亡和现场封锁状态。
2. **查看现场图**：建筑轮廓和区域边界已知，格子内容未知；少量格子可作为初始已揭示事实。
3. **获取现场定则**：搭档进行定则感知，公司设备整理为本关全部公开规则。
4. **循环推理与调查**：寻找必然安全或必然异常的格子；调查安全格、标记异常格、读取可选资料。
5. **完成全场勘察**：所有安全格必须手动调查，所有异常格必须正确标记。
6. **提交完整现场登记图**：专业处置单位接管，异常可在成功结算后有限展示。
7. **自动结案**：系统依据已登记内容生成理赔摘要。

> **完成条件：全部安全区域已调查 + 全部异常区域已正确标记。**

## 11. 玩家动作与界面状态

| 动作／状态 | 行为要求 |
|---|---|
| 调查未知格 | 安全则揭示；异常则立即失败。 |
| 标记异常 | 纯笔记状态，可取消；不触发世界变化。 |
| 标记安全 | 纯笔记状态，可取消；用于安排下一次调查。 |
| 查看规则范围 | 高亮规则涉及的有效邻域。 |
| 请求提示 | 返回一个当前可证明的必然结论及简要原因。 |
| 提交现场图 | 检查完成条件；错误提交不得泄露具体错误格。 |
| 未知格 | 只显示区域边界与统一遮蔽，不显示可推断内容的视觉差异。 |
| 已揭示安全格 | 显示内容图标、现场细节及可能的资料入口。 |
| 异常格 | 正常游玩阶段不直接揭示；成功结算后可展示。 |

## 12. 现场信息与碎片叙事

### 12.1 定则对象

参与格子规则的内容，例如应急灯、尸体、终端、档案柜、医疗设备和空区域。必须使用统一图标，并在规则中被明确点名。尸体只有在本关规则明确提及时才参与推理。

### 12.2 现场记录

服务于当前案件的小叙事与定损，例如尸体身份、设备损坏、维修日志、聊天记录、货物清单和事故录音。它们不必构成另一套完整案件谜题，也不得暗示异常格坐标。

### 12.3 关联资料

调查某种物品或现象后，公司数据库可以匹配出旧赔案、旧版手册、能力测试记录或内部通知。关联资料不必与当前案件直接相关，不要求汇聚成宏大主线。

> **档案文本不构成现场定则判定依据。**

## 13. 保险身份与自动结案

保险职业是玩家调查全部安全格的自然理由，也是游戏的叙事语气来源。当前版本不将“是否理赔”设计为玩家必须解决的第二套推理题。

自动报告可包含：核定人员伤亡、核定财产损失、异常直接损失是否成立、是否发现未申报设备、赔案进入赔付或特别审核程序。

示例：`核定人员伤亡：7｜核定财产损失：1,840,000｜异常直接损失：成立｜赔案状态：进入赔付流程`

将来只有在某些案件天然适合且不增加过多关卡约束时，才考虑加入轻量理赔选择。该功能属于备选扩展。

## 14. 首案：《别按那个键》

**定位：Game Jam 主题案件 + 教学关。该案的黑盒和红色按键只属于本案。**

### 14.1 案件概况

一家小型交互设备公司发生严重人员伤亡。客户申报办公设备损失、营业中断和雇主责任赔偿。初步勘察发现现场内容受到遮蔽，常规调查中止。幸存者留下的最后一句有效录音是：“别按那个键。”

### 14.2 异常

异常是若干个没有接缝和标识的哑黑色盒子，表面只有一枚红色按键。任何人对盒子建立主动观察，都会迅速产生无法抵抗的确定感：按下按钮是当前唯一合理的操作。

按钮被按下后，现场范围内所有生命活动会被瞬间抹除，人员以尸体或失活遗留物的形式留在现场。保险调查员因此绝不能直接调查盒子所在区域。

### 14.3 可用安全内容

- 应急灯。
- 测试终端。
- 员工尸体。
- 空办公区。
- 产品原型、会议桌、快递包装等叙事物件。

### 14.4 可回收资料

- 没有寄件人的样机签收单。
- 员工讨论按键用途的聊天记录。
- 负责人要求继续测试的邮件。
- 中途切断的监控音频。
- 写着“按键测试不得由单人完成”的便签。

这些资料只构成短小事故故事，不参与异常坐标推理。

### 14.5 教学目标

未知格、安全调查、异常总数、正交邻域、邻接域、安全／异常标记、规则范围、提示、调查全部安全格、提交完整现场图。

### 14.6 Game Jam 可拆卸性

- 《别按那个键》只存在于案件标题、异常外形和少量文本中。
- 它不承担世界观基础、搭档能力来源或全作主线。
- 赛后可更名、替换异常、改为附加案件或整体移除。

| 《无主病床》 | 私人诊所中出现没有对应患者的病床；主动确认患者身份的人会成为新的住院记录。 |
| 《最后一席》 | 晚宴结束后，尸体数量比宾客名单多一人；异常藏在无法核实身份的座席中。 |
| 《无灯楼层》 | 商业大厦出现不在图纸上的楼层；调查错误房门会使勘察人员被登记为长期租户。 |
| 《未放映母带》 | 电影资料馆中混入不存在于目录的胶片；读取任意一帧会使画面事件立即发生。 |
| 《缺席货舱》 | 轨道运输船多出无货运编号的舱室；开启后乘员总数会永久增加。 |
| 《重复遗体》 | 同一具尸体出现在多个区域记录中；确认错误的那一具会让所有在场人员共享同一身份。 |

## 16. 美术、UI 与声音方向

- 主界面以拟真的平面图、CAD 图、消防图和保险公估软件为基础。
- 未知区域只显示边界、编号和统一遮蔽，不出现能泄露内容类型的细微差异。
- 安全调查后，图纸补上内容图标、财产符号、尸体标记和现场注释。
- 定则对象使用标准图标；叙事物件使用独立勘验图层。
- 异常在正常推理阶段默认不可见，只在成功结算或后续处置影像中出现。
- 搭档语音是提示、规则确认和角色关系的主要载体。
- 环境音只制造现场感，不得依据异常位置发生可推理变化。

## 19. 程序状态与关键验收条件

### 19.1 格子状态

- `unknown`
- `marked_safe`
- `marked_anomaly`
- `revealed_safe`

异常格在正常游玩阶段不进入 `revealed_anomaly` 状态；它们只在成功结算展示，或在失败演出中结束本次尝试。

### 19.2 成功校验

- 每个安全格都处于 `revealed_safe`。
- 每个异常格都处于 `marked_anomaly`。
- 没有安全格被标记为异常。
- 提交失败时不显示哪一格错误。

### 19.3 提示校验

- 提示必须基于当前公开规则与已揭示事实。
- 提示只能指向所有候选现场中一致的结论。
- 优先给出可立即行动的必然安全格；若没有，再给出必然异常格。
- 提示文本应引用相关定则，不能只说“答案是 C3”。

## 20. MVP／Game Jam 范围建议

- 仅实现《别按那个键》。
- 3×3 或 4×4 棋盘。
- 3 种核心安全类型 + 空区。
- 2 至 3 个异常格。
- 4 至 6 条规则。
- 3 至 5 份短资料。
- 自动理赔摘要 + 黑盒揭示图。
- 一套快速的按键灾害失败演出。

首版重点是验证：玩家是否理解“规则可靠、未知内容必须调查、异常绝不能触碰、全部安全格必须清完”这四件事。

## 21. 当前明确不做／暂缓的内容

- 不设置贯穿全作的统一黑盒异常。
- 不设置“异常依赖信息缺口才能存在”的复杂理论。
- 不让玩家亲自收容、消灭或测试异常。
- 不设计污染值、精神值、战斗和即时追逐。
- 不让搭档给出错误规则，也不要求玩家判断规则真假。
- 不把剧情文本、音效或视觉异样当作隐藏位置线索。
- 不强制设计保险责任选择或完整剧情谜案；当前由程序自动结案。
- 不要求所有单元案件最终归并为同一阴谋。

## 22. 术语表

| 术语 | 定义 |
|---|---|
| 未登记现场 | 区域内容受到异常信息遮蔽、需要非常规保险勘察的事故现场。 |
| 现场信息遮蔽 | 使目击者与常规扫描无法确认每个区域具体内容的异常效应。 |
| 现场定则 | 搭档感知并经设备标准化后，公开给玩家的精确关系规则。 |
| 定则感知 | 搭档感知“物与物之间必然关系”的特殊能力。 |
| 安全格 | 可以主动调查且不会触发异常灾害的区域。 |
| 异常格 | 异常所在区域；主动调查立即失败，只能推理并标记。 |
| 现场记录 | 服务于当前案件的小叙事、定损和氛围资料，不参与格子判定。 |
| 关联资料 | 由公司数据库匹配出的旧案件或内部档案，不必与当前案件直接相关。 |
| 候选现场集合 | 满足全部规则和已知事实的所有可能完整布局。 |

## 23. 最终核心陈述

玩家和搭档来到一个内容被异常遮蔽的事故现场。搭档能够感知可靠的空间定则，玩家据此逐格调查全部安全区域，登记尸体、财产与资料；异常只能被定位，绝不能被主动调查。每一关是一宗独立赔案，异常、地点和小故事都可以更换，而公平的固定信息推理始终不变。
