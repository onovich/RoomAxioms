# 《房间公理》UI / UX 设计文档

**版本：** 0.1  
**目标：** 为浏览器 MVP 提供可实现、可测试的界面规范

---

## 1. 设计目标

界面首先服务于逻辑比较，其次才营造怪谈氛围。玩家应当能够在不抄写纸笔的情况下完成以下工作：

- 随时看到全部规则；
- 清楚知道每条规则的精确范围和数量词；
- 快速区分未知、已揭示、访客标记和安全笔记；
- 查看已获得物证与最近行动；
- 在需要时得到可解释提示，而非直接给答案；
- 明白失败来自哪次调查，而不是来自隐藏规则。

### 1.1 设计原则

1. **规则常驻**：桌面端规则面板默认不折叠。
2. **机械语义优先**：风味文本不抢占精确规则的视觉层级。
3. **范围可视化**：任何“正交邻域/邻接域”都可一键在棋盘上高亮。
4. **颜色非唯一编码**：危险、安全、笔记均有图形、文字或纹理冗余。
5. **玩家模式与开发者模式分离**：候选数和强制格默认不向玩家泄露。
6. **不制造伪时间压力**：MVP 没有倒计时。

## 3. 桌面调查主界面

### 3.1 布局

推荐最小桌面宽度 1180 px。主界面为三栏：

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│ 房间公理 / 案卷 04          访客 0/2     已调查 3/16     [提示] [设置]      │
├──────────────────────┬───────────────────────────────┬──────────────────────┤
│ 规则与术语            │            棋盘               │ 证据与笔记            │
│                      │                               │                      │
│ R1 全局数量           │        A   B   C   D          │ 最近揭示              │
│ R2 全局数量           │    1  [?][瓶][?][?]          │ B1：酒瓶              │
│ R3 酒瓶→垃圾桶        │    2  [瓶][?][瓶][?]          │ A2：酒瓶              │
│ R4 垃圾桶→安全域      │    3  [?][?][?][?]          │ C2：酒瓶              │
│ R5 镜子→访客计数      │    4  [?][?][?][?]          │                      │
│ R6 酒瓶→无访客        │                               │ 当前工具              │
│                      │    [调查] [标访客] [标安全]    │ ○ 调查                │
│ [邻域定义]            │                               │                      │
├──────────────────────┴───────────────────────────────┴──────────────────────┤
│ 状态：全部规则已公开。调查只会揭示客观物件。          [重置] [提交结论]     │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 栏宽建议

- 左侧规则栏：320-360 px；
- 中间棋盘：剩余弹性空间，最大 720 px；
- 右侧证据栏：280-320 px；
- 栏间距：20-24 px；
- 内容最大宽度：1440 px，居中。

### 3.3 顶栏

必须显示：

- 项目/关卡名；
- 已标记访客数量 / 公开总数；
- 已调查格数 / 总格数；
- 提示入口；
- 设置入口。

不显示候选世界数，除非开发者模式开启。

## 4. 棋盘与格子组件

### 4.1 尺寸

- 4×4：桌面单格 88-104 px；
- 5×5：桌面单格 72-88 px；
- 平板单格不得小于 56 px；
- 触控命中区域不得小于 44×44 px。

### 4.2 格子状态

| 状态 | 视觉 | 交互 |
|---|---|---|
| 未知 | 深色封板、坐标角标、细纹理 | 可调查或标记 |
| 悬停 | 边框提亮、轻微上浮 | 显示坐标与当前工具 |
| 键盘聚焦 | 2-3 px 高对比焦点环 | Enter 执行当前工具 |
| 已揭示空地 | 平面底色、中心点或“空”标签 | 不可再次调查 |
| 已揭示物件 | 图标 + 文字短标签 | 悬停可显示受哪些规则约束 |
| 访客标记 | 旗形/人形轮廓 + “访客？” | 可撤销，不判定真伪 |
| 安全笔记 | 圆环/铅笔纹 + “安全？” | 可撤销 |
| 规则范围高亮 | 外描边或斜纹 | 仅说明范围，不表示安全 |
| 开发者强制安全 | 绿色角标 S | 只在开发者模式 |
| 开发者强制危险 | 红色角标 G | 只在开发者模式 |
| 触发失败 | 遮罩破裂、显示访客 | 进入失败态 |

### 4.3 图标原则

每种物件同时提供：

- 图形轮廓；
- 中文短标签；
- 屏幕阅读器文本；
- 非颜色差异。

建议图标：

- 酒瓶：细颈瓶；
- 垃圾桶：带盖容器；
- 镜子：竖向椭圆/矩形镜面；
- 访客标记：空心人形或门缝轮廓；
- 空地：小圆点或“空”。

### 4.4 鼠标与触控

桌面默认：

- 左键：调查；
- 右键：在“访客标记 → 安全笔记 → 无标记”之间循环；
- Shift + 左键：直接切换安全笔记；
- 鼠标中键不使用。

触控：

- 先选择底部工具，再点格子；
- 长按可弹出“调查 / 标访客 / 标安全”；
- 不依赖双击。

为避免误触，已标记访客的格子再次调查时要求一次确认。

## 5. 规则卡

### 5.1 信息层级

每张规则卡包含：

1. 编号与图标；
2. 一句风味文本，可折叠；
3. 加粗的精确机械句；
4. 范围徽标，例如“正交邻域”；
5. 数量徽标，例如“恰好 1”；
6. “在棋盘上查看范围”按钮。

示例：

```text
R5  镜面登记
镜面中只能出现一位未登记住客。

每面镜子的【邻接域】中，【恰好 1】名访客。
[邻接域：角3 / 边5 / 内8]   [查看范围]
```

### 5.2 单向关系提示

为降低“访客有两个，所以镜子也有两个”这类误读，规则卡使用箭头方向：

```text
镜子  ──约束其邻接域──>  访客数量 = 1
```

并在术语帮助中明确：

> 规则只从左侧主体约束右侧目标，不自动推出反向关系。

### 5.3 高亮行为

点击规则卡后：

- 若没有选中主体格，显示邻域示意图；
- 若玩家再点一格，临时把该格视为主体并高亮对应范围；
- 若点到已揭示的对应主体物件，规则卡同时列出其当前已知计数；
- 高亮颜色不使用安全/危险色，避免误导。

---

## 6. 邻域说明弹层

必须以图示而非仅文字解释边缘语义。

### 6.1 正交邻域图示

```text
角落（2）    边缘（3）    内部（4）
[X][·][ ]    [·][X][·]    [ ][·][ ]
[·][ ][ ]    [ ][·][ ]    [·][X][·]
[ ][ ][ ]    [ ][ ][ ]    [ ][·][ ]
```

### 6.2 邻接域图示

```text
角落（3）    边缘（5）    内部（8）
[X][·][ ]    [·][X][·]    [·][·][·]
[·][·][ ]    [·][·][·]    [·][X][·]
[ ][ ][ ]    [ ][ ][ ]    [·][·][·]
```

图例：`X` 为主体格，`·` 为纳入规则的棋盘内格子。棋盘外永远不计入数量，也不被假定为空格。

## 7. 证据与笔记栏

### 7.1 证据日志

按时间倒序显示：

- 坐标；
- 揭示物件；
- 与哪些规则有关；
- 可选“聚焦棋盘”按钮。

日志只记录事实，不记录系统对安全/危险的隐式判断。

### 7.2 玩家笔记

显示：

- 已标访客坐标；
- 已标安全坐标；
- 标记数量与公开总数是否匹配；
- 一键清除全部铅笔笔记。

### 7.3 推理草稿（后续）

非 MVP 可加入“选中两条规则，查看范围交集”工具。MVP 不应自动解题，但可允许玩家临时锁定多个范围高亮。

## 8. 提示系统

### 8.1 三层提示

1. **方向提示**：指出应关注的规则或已知物证，不给坐标。
2. **范围提示**：高亮相关格和候选集合。
3. **结论提示**：给出一个强制安全或强制危险格及完整证明。

MVP 至少实现第 3 层，证明必须由人类推理器生成。

### 8.2 证明卡格式

```text
结论：B2 可以安全调查。

前提：
- 房间中恰有 1 个垃圾桶。（R1）
- 每只酒瓶的正交邻域中恰有 1 个垃圾桶。（R3）
- 已知酒瓶位于 B1、A2、C2。

推理：
三个正交邻域的共同候选只有 B2。
因此 B2 必为垃圾桶；垃圾桶不是访客，所以 B2 安全。
```

提示后只高亮结论，不自动执行调查。

## 9. 成功与失败界面

### 9.1 成功结算

显示：

- 访客位置；
- 调查次数；
- 无信息调查次数；
- 提示次数；
- 本关关键推理模板；
- 可展开的完整证明链；
- 重玩、下一关、返回案卷。

### 9.2 失败结算

显示：

- 触发访客的坐标；
- “规则没有变化；本次调查在当时并非必然安全”的提示；
- 可选择重开或查看一步纠错提示；
- 默认不立即揭示全部访客，避免失败成为探测工具。开发模式可显示。

### 9.3 错误提交

MVP 文案：

> 当前标记与公开规则仍存在冲突，或尚不能唯一覆盖全部访客。你可以继续修正。

不得指出具体错误标记。

## 10. 开发者验证层

开发者模式需明显标注“非玩家信息”。建议右栏增加：

- 当前候选世界数（可设上限并显示 `>10000`）；
- 强制安全格列表；
- 强制访客格列表；
- 危险布局是否唯一；
- 人类推理器可用动作；
- 最近一次查询耗时；
- 目标棋盘显示；
- 当前状态导出 JSON；
- 证明图节点。

棋盘覆盖层使用：

- `S`：精确求解器强制安全；
- `G`：精确求解器强制访客；
- `H`：人类推理器可以解释；
- `!`：精确可证但解释器缺失。

## 11. 响应式设计

### 11.1 平板 768-1179 px

- 规则栏变为左侧可固定抽屉；
- 证据栏变为右侧抽屉或底部标签；
- 棋盘保持主视觉；
- 顶栏保留进度、提示、设置。

### 11.2 手机 <768 px

布局分为三个底部标签：

1. 棋盘
2. 规则
3. 证据

棋盘页底部固定工具条。规则页必须保留邻域图示。手机只作为兼容目标，首轮试玩仍以桌面/平板为准。

### 11.3 横竖屏

- 竖屏优先纵向滚动规则；
- 横屏可恢复左右双栏；
- 棋盘不得因横向溢出而缩小到低于触控最小尺寸，可允许局部缩放或滚动。

## 12. 键盘与无障碍

### 12.1 键盘映射

- 方向键：移动棋盘焦点；
- Enter / Space：调查；
- F：切换访客标记；
- S：切换安全笔记；
- R：聚焦规则栏；
- H：打开提示；
- Esc：关闭弹层或清除临时范围高亮；
- Ctrl/Cmd + R 不重载游戏状态，浏览器默认行为不拦截。

### 12.2 屏幕阅读器

格子 accessible name 示例：

- `B2，未知，未标记`；
- `C1，已揭示，镜子`；
- `D1，未知，玩家标记为访客`。

规则卡使用列表语义；数量词和范围不得只放在 tooltip 中。

### 12.3 视觉

- 正文字号至少 16 px；
- 规则精确句至少 16-18 px；
- 对比度目标 WCAG AA；
- 支持减少动态效果；
- 支持 200% 浏览器缩放；
- 不使用闪烁和突发白屏。

## 13. 视觉系统

### 13.1 色彩令牌

| Token | 建议值 | 用途 |
|---|---|---|
| `--bg` | `#0B0F0E` | 页面背景 |
| `--surface-1` | `#141A18` | 面板 |
| `--surface-2` | `#1C2421` | 卡片/未知格 |
| `--text` | `#EFECE3` | 主文字 |
| `--muted` | `#A8B0AA` | 次要文字 |
| `--accent` | `#D5A84E` | 选中与关键动作 |
| `--safe` | `#72B493` | 安全笔记/开发提示 |
| `--danger` | `#D76B68` | 危险与失败 |
| `--info` | `#7AA6C8` | 范围说明 |
| `--border` | `#34403B` | 分隔线 |

这些值是原型基线，不是最终品牌定稿。

### 13.2 字体

- 中文：`Noto Sans SC`、`PingFang SC`、`Microsoft YaHei`、系统无衬线；
- 坐标和调试数据：等宽字体；
- 不使用难读的手写恐怖字体承载规则正文。

### 13.3 动效

- 揭示：120-180 ms 的翻板/淡入；
- 标记：80-120 ms；
- 规则范围：轻微脉冲一次后静止；
- 失败：不超过 500 ms，且支持 reduced-motion。

## 15. 文案基线

- 开局承诺：**“全部规则已公开。调查只会揭示房间中早已存在的物件。”**
- 未知格辅助文案：**“尚未调查。”**
- 访客标记：**“玩家判断：此处可能是访客。”**
- 安全笔记：**“玩家判断：此处计划安全调查。”**
- 提示入口：**“解释一个必然结论”**，避免“给答案”。
- 开发模式：**“验证层已开启：以下信息不属于玩家可见线索。”**

## 16. 原型说明

随交接包提供 `prototype/index.html`，包含：

- 三栏桌面布局和响应式变化；
- 基准样例棋盘；
- 调查、右键标记、工具切换、提交、提示和重置；
- 邻域说明弹层；
- 开发者验证层；
- 样例级候选世界枚举与强制格显示。

该 HTML 用于验证信息架构与交互，不代表最终 React 代码结构。
