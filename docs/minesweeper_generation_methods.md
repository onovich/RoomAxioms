# 扫雷（Minesweeper）关卡生成方法论整理

本文总结多种扫雷关卡生成方法，从基础随机生成到约束求解与进化算法，并给出工程化建议。

---

## 1. 普通随机生成（Naive Random Generation）

### 方法
1. 在棋盘中随机布雷（避开首次点击区域）
2. 根据雷位置计算数字
3. 开始游戏

### 特点
- 实现简单
- 速度极快
- 无法保证“无猜可解”

### 问题
- 容易产生必须猜测的局面（50/50）
- 难度不可控

---

## 2. Las Vegas 生成-测试法（推荐基础方案）

### 核心思想
> 随机生成 + 严格验证 + 不合格就丢弃

### 流程
1. 随机生成雷分布
2. 计算数字盘
3. 使用扫雷求解器尝试“无猜解题”
4. 若失败则丢弃
5. 若成功则根据难度评分筛选

### 优点
- 可以保证“可解性”
- 可控难度分布
- 工程上最常用

### 伪代码
```
repeat:
    board = random_board()
    if solver(board).no_guess_success:
        return board
```

---

## 3. 规则推理求解器 + 生成

### 方法
使用人类基础规则模拟扫雷解题过程：

#### 规则
- 数字 = 周围雷数
- 已标雷 = 减少约束
- 若唯一剩余 → 推断安全/雷

### 优点
- 模拟真实玩家体验
- 可用于“Easy/Medium”生成

### 缺点
- 推理能力有限
- 复杂局面无法处理

---

## 4. CSP / SAT 约束求解法（高级）

### 核心思想
将扫雷建模为约束系统：

```
x_i ∈ {0,1}
Σ邻域 x_i = 数字
```

### 方法
- 把未知格子当变量
- 每个数字格是约束方程
- 用 SAT / CSP 求解一致性

### 用途
- 判断“是否必须猜”
- 提供高质量难度评估
- 用于 Expert 级生成

### 优点
- 严格
- 可证明无猜

### 缺点
- 实现复杂
- 性能较重

---

## 5. 反向构造法（Constraint Construction）

### 方法
1. 先设计解题路径
2. 再构造满足该路径的雷布局

### 优点
- 可以控制解题体验
- 适合教学关卡

### 缺点
- 容易产生非自然布局
- 难保证无捷径解法

---

## 6. 遗传算法（Genetic Algorithm）

### 核心思想
模拟进化：

- 选择（selection）
- 交叉（crossover）
- 变异（mutation）

### 编码方式
- 雷的位置集合
或
- 二进制棋盘

### 适应度函数示例
```
fitness =
    + 无猜可解
    + 难度匹配目标
    + 展开区域大小
    - 50/50 猜测惩罚
    - 过简单惩罚
```

### 优点
- 可优化“体验质量”
- 可生成多样性

### 缺点
- 不能单独保证可解性
- 需要配合求解器

---

## 7. BFS / 搜索方法（辅助用）

### BFS用途
- 展开 0 区域（flood fill）
- 模拟局部状态扩展

### 不适用
- 不适合作为完整扫雷求解器
- 状态空间爆炸

---

## 8. 推荐工业级方案（最佳实践）

### 组合方案

```
随机生成（Las Vegas）
+ 基础规则推理
+ CSP/SAT 边界检查
+ 难度评分系统
```

### 流程

1. 随机生成雷
2. 避开 first click 安全区
3. 计算数字
4. 用规则推理解题
5. 卡住 → CSP/SAT 判断
6. 是否无猜可解
7. 评分
8. 返回

---

## 9. 难度设计建议

### Easy
- 仅基础规则即可解决

### Medium
- 需要局部组合推理

### Hard
- 需要子集推理 / 边界推理

### Expert
- CSP/SAT 推理参与

---

## 10. 总结

扫雷生成本质是：

> “随机生成 + 可解性验证 + 难度控制”

核心技术路径：

- Las Vegas（随机 + 验证）
- 约束求解（CSP / SAT）
- 规则模拟（人类推理）
- 可选：遗传算法优化体验
